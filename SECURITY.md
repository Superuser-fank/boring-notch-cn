# 安全政策

Boring Notch CN 是 Boring Notch 的非官方 fork。本仓库只处理这个 fork 自身引入或打包分发相关的安全问题。

## 如何报告

请优先使用 GitHub Security Advisory。如果只能通过 Issue 反馈，请不要在公开内容中直接贴可利用细节、凭据、Token、证书或个人隐私数据。

- 仓库：<https://github.com/Superuser-fank/boring-notch-cn>
- Issues：<https://github.com/Superuser-fank/boring-notch-cn/issues>
- Security Advisories：<https://github.com/Superuser-fank/boring-notch-cn/security/advisories>

## 范围

适合在本仓库报告：

- Boring Notch CN 自身代码导致的权限、隐私或本地安全问题。
- 本 fork 的签名、打包、Release 分发或更新流程问题。
- 本 fork 引入的第三方依赖使用方式问题。

不适合在本仓库报告：

- 上游 Boring Notch 原项目问题。请报告给上游维护者。
- 网易云音乐、QQ 音乐、Apple Music、Spotify、YouTube Music、微信、QQ、钉钉、飞书等第三方服务自身问题。请报告给对应服务维护方。
- macOS 系统漏洞。请报告给 Apple。

## 隐私说明

媒体诊断信息设计为不包含歌曲名、歌手名、联系人或文件内容。提交 Issue 前仍建议检查日志和截图，移除 Apple ID、本地用户名、设备序列号、访问令牌等敏感信息。
